# 高亮功能测试步骤

## 测试环境
- 浏览器: 推荐使用 Chrome 或 Edge
- 开发服务器: http://localhost:5174/
- 需要打开浏览器控制台查看调试日志

## 测试步骤

### 1. 启动项目
```powershell
cd "d:\Citation Visualization\paper"
npm run dev
```

### 2. 打开浏览器
访问 http://localhost:5174/

### 3. 打开开发者工具
按 `F12` 或 `Ctrl+Shift+I` 打开浏览器控制台

### 4. 切换到可视化视图
点击页面上的 "引用关系可视化" 按钮

### 5. 测试高亮功能

#### 测试用例 1: 点击连接线
1. 在可视化图中找到任意一条连接线
2. 点击该连接线
3. **预期结果**:
   - 连接线变粗且高亮 (stroke-width: 4, opacity: 1)
   - 左侧论文内容区域自动滚动到相关引用位置
   - 引用标记 (如 `[1]`, `[24]` 等) 显示为**红色背景**
   - 包含该引用的整个段落显示为**淡黄色背景**
   - 控制台输出相关日志

#### 测试用例 2: 切换不同连接线
1. 点击另一条连接线
2. **预期结果**:
   - 之前的高亮被清除
   - 新的引用被高亮
   - 页面滚动到新的位置

#### 测试用例 3: 多个引用在同一段落
1. 找一条连接两个相邻引用ID的连接线
2. 点击该连接线
3. **预期结果**:
   - 同一段落中的两个引用都会被高亮
   - 段落背景只显示一次

### 6. 查看控制台日志

应该看到类似以下的日志输出:
```
高亮引用: [1, 24]
找到匹配的引用: [1-2]
找到匹配的引用: [24]
已添加高亮类
已添加高亮类
滚动到高亮位置
```

## 常见问题排查

### 问题 1: 点击连接线无反应
**检查项**:
- 控制台是否有错误信息?
- 是否有 "高亮引用:" 日志输出?
- Emitter 是否正常工作?

**解决方法**:
检查 Visualization.vue 中的事件发送代码是否正确

### 问题 2: 有日志但没有高亮
**检查项**:
- 是否有 "找到匹配的引用:" 日志?
- 是否有 "已添加高亮类" 日志?
- 检查页面元素是否有 `.highlight-ref` 和 `.has-highlight` 类

**解决方法**:
1. 打开浏览器开发者工具的 Elements 面板
2. 搜索 `highlight-ref` 类
3. 检查 CSS 样式是否正确加载

### 问题 3: 高亮显示但不滚动
**检查项**:
- 是否有 "滚动到高亮位置" 日志?
- `.highlight-ref` 元素是否存在?

**解决方法**:
检查 `scrollIntoView` 方法是否被调用

### 问题 4: 引用ID不匹配
**检查项**:
- Visualization 组件发送的 ID 是否正确?
- 匹配逻辑是否正确处理不同格式?

**解决方法**:
在控制台中手动测试匹配逻辑

## 手动测试代码

在浏览器控制台中运行以下代码来测试:

```javascript
// 测试查找 sup 标签
const supTags = document.querySelectorAll('.research-paper sup');
console.log('找到的 sup 标签数量:', supTags.length);

// 测试匹配逻辑
supTags.forEach(sup => {
  console.log('引用内容:', sup.textContent);
});

// 测试高亮一个具体的引用
const testHighlight = (id) => {
  supTags.forEach(sup => {
    const text = sup.textContent.trim();
    const singleMatch = text.match(/^\[(\d+)\]$/);
    if (singleMatch && parseInt(singleMatch[1]) === id) {
      sup.classList.add('highlight-ref');
      sup.closest('p').classList.add('has-highlight');
      console.log('已高亮引用:', text);
    }
  });
};

// 测试高亮引用 [1]
testHighlight(1);
```

## 视觉效果检查

### 引用标记样式
- 背景: 红色到粉红色渐变 (#ff6b6b → #ee5a6f)
- 文字颜色: 白色
- 内边距: 2px 6px
- 圆角: 4px
- 粗体显示
- 阴影效果
- 脉冲动画 (0.6秒)

### 段落背景样式
- 背景: 淡黄色渐变 (半透明)
- 内边距: 15px
- 圆角: 8px
- 左边框: 4px 黄色 (#fdcb6e)
- 阴影效果
- 渐入动画 (0.6秒)

## 性能检查

- 高亮操作应该在 100ms 内完成
- 滚动动画应该流畅 (smooth)
- 不应该有页面卡顿或闪烁

## 浏览器兼容性

测试通过的浏览器:
- [ ] Chrome 120+
- [ ] Edge 120+
- [ ] Firefox 120+
- [ ] Safari 17+

## 已知限制

1. 目前只高亮 `<sup>` 标签本身,不高亮整个句子
2. 如果一个段落有多个匹配的引用,会添加多个 `.highlight-ref` 类
3. 脚注引用 (如 `①`) 可能不会被匹配

## 后续优化建议

1. 可以考虑高亮整个句子而不仅仅是 sup 标签
2. 可以添加淡出效果来清除高亮
3. 可以添加引用标记的脉冲动画来吸引注意力
4. 可以在段落中用特殊颜色标记匹配的句子边界
